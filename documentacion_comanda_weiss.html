Diseño de Catálogo y Comandas

1. Estructura del Catálogo y Promociones

Teniendo en cuenta que existen productos compuestos (como una hamburguesa) y que es en realidad una receta compuesta por otros ítems de inventario (insumos). Proponemos clasificar los ítems en cuatro tipos:

  • Item de Venta Final: El producto que el cliente ordena y paga. Su disponibilidad depende del stock de sus componentes.
  • Item de Insumo (Materia Prima): El ingrediente básico comprado a proveedores (carne, pan, queso).
  • Item de Packaging: Materiales no comestibles para la entrega (cajas, servilletas).

Explicación y Uso

  • Doble Funcionalidad: Este ticket tiene dos propósitos clave. Primero, sirve como recibo detallado para el cliente, permitiéndole verificar su compra. Segundo, funciona como una lista de chequeo (checklist) para el personal de entrega, que lo utiliza para ensamblar correctamente todos los productos del pedido final.
  • Organización Jerárquica: La información del ticket se estructura de forma intuitiva. Los productos se agrupan bajo dos posibles encabezados: "PROMO" para los artículos en promoción, y "PRODUCTOS" para cualquier artículo comprado de forma individual.
  • Contenido Exhaustivo: El ticket muestra la totalidad del pedido, incluyendo tanto los productos que se preparan en la cocina como los que no (bebidas, postres), garantizando que sea un comprobante completo y una guía precisa para la entrega.
  • Visualización de Modificaciones: Si un cliente solicita quitar un ingrediente estándar de un producto (por ejemplo, una hamburguesa "sin cebolla"), esta modificación se indica de forma clara y visible directamente debajo del nombre del producto afectado.
  • Registro de Observaciones: Cualquier solicitud especial del cliente que no sea la eliminación de un ingrediente (como "carne bien cocida") también se detalla debajo del producto correspondiente, asegurando que la cocina reciba todas las instrucciones.
  • Proceso de Impresión: El ticket se genera e imprime una única vez desde el Punto de Venta (POS) en el momento exacto en que se confirma y cierra la transacción.
  • Agrupación de Ítems Idénticos: Para optimizar la lectura y reducir errores, los productos idénticos se agruparán con un multiplicador de cantidad (ej: 2x Burger Roger). Esta regla solo se aplica si los productos son exactamente iguales. Si uno de ellos tiene una modificación o una observación especial (ej: sin cebolla), deberá listarse siempre en una línea separada para garantizar que la personalización sea visible y se ejecute correctamente.

4. Visualización del KDS (Kitchen Display System)

Explicación y Uso

  • Agrupación de Ítems Idénticos: Los productos idénticos se agruparán con un multiplicador de cantidad (ej: 2x Burger Roger). Esta regla solo se aplica si los productos son exactamente iguales. Si uno de ellos tiene una modificación o una observación especial (ej: sin cebolla), deberá listarse siempre en una línea separada para garantizar que la personalización sea visible y se ejecute correctamente.

    • Esta es una maqueta de cómo se vería una orden en la pantalla del KDS. Cada orden es una "tarjeta" interactiva. El encabezado cambia de color según el tiempo transcurrido para una rápida identificación visual del estado del pedido.

    • Verde (0-10 min): Pedido nuevo, dentro del tiempo esperado.
    • Amarillo (10-20 min): El pedido se acerca al límite de tiempo.
    • Rojo (>20 min):Pedido demorado.

    El botón "LISTO" en cada tarjeta permitiría al cocinero marcar la orden como completada, quitándola de la pantalla activa.
